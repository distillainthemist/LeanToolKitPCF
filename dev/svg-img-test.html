svgExport → <img> render test

The DOM cards' svgExport uses SVG <foreignObject>. It renders fine as a document, but some Safari versions refuse it inside an <img> — which is how a board template would show card tiles. Open this file in Safari, paste a card's svgExport value below (from the harness output panel), and press Test. If the left pane shows the card, Safari is good; if it stays blank while the right pane (inline SVG) shows it, boards must fall back to pngExport for tiles.
